# WordCards 架构设计文档

## 1. 系统概述

WordCards 是一个基于间隔重复算法（SRS）的单词记忆应用，采用前后端分离架构，支持离线优先的用户体验。

### 1.1 设计目标

| 目标 | 描述 |
|------|------|
| **离线优先** | 无需用户登录，使用设备 ID 标识用户 |
| **高性能** | 支持 10000+ 单词流畅滚动 |
| **可扩展** | 支持多词库、自定义词库导入 |
| **易部署** | 支持 Docker、云平台多种部署方式 |

### 1.2 技术选型

```
┌─────────────────────────────────────────────────────────────┐
│                        前端技术栈                            │
├─────────────────────────────────────────────────────────────┤
│  React 19 + TypeScript + Vite                               │
│  ├── 状态管理：React Hooks + localStorage                   │
│  ├── 路由管理：Hash Router（SPA）                           │
│  ├── 性能优化：@tanstack/react-virtual                      │
│  └── 样式方案：CSS Variables + 内联样式                     │

└─────────────────────────────────────────────────────────────┘
```

## 3. 核心模块设计

### 3.1 SRS 算法模块

间隔重复算法（Spaced Repetition System）是本应用的核心，基于 SM-2 算法改进。

#### 算法流程

```
┌─────────────────────────────────────────────────────────────┐
│                      SRS 算法流程                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  用户反馈                                                    │
│     │                                                       │
│     ├─── "不认识" ──────────────────────────────┐          │
│     │     下次复习：10分钟后                     │          │
│     │     ease = max(1.3, ease - 0.2)           │          │
│     │     streak = 0                            │          │
│     │                                           │          │
│     ├─── "模糊" ────────────────────────────────┤          │
│     │     下次复习：1.2 × 当前间隔               │          │
│     │     ease = max(1.3, ease - 0.05)          │          │
│     │     streak = 0                            │          │
│     │                                           │          │
│     └─── "认识" ────────────────────────────────┘          │
│           下次复习：ease × 当前间隔                          │
│           ease = min(2.8, ease + 0.05)                      │
│           streak += 1                                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 核心参数

| 参数 | 范围 | 初始值 | 描述 |
|------|------|--------|------|
| ease | 1.3 - 2.8 | 2.3 | 难度因子，越高复习间隔越长 |
| intervalDay | 0+ | 0 | 间隔天数 |
| streak | 0+ | 0 | 连续认识次数 |
| reviews | 0+ | 0 | 总复习次数 |

#### 代码实现

```typescript
export function 计算下一次复习(
  current: 复习状态,
  rating: 记忆反馈,
  now = new Date(),
): 复习状态 {
  const baseEase = current.ease
  const reviews = current.reviews + 1

  if (rating === '不认识') {
    return {
      ...current,
      dueAt: new Date(now.getTime() + 10 * 60 * 1000),
      intervalDay: 0,
      ease: Math.max(1.3, baseEase - 0.2),
      streak: 0,
      reviews,
    }
  }

  if (rating === '模糊') {
    const nextInterval = Math.max(1, Math.round((current.intervalDay || 1) * 1.2))
    return {
      ...current,
      dueAt: new Date(now.getTime() + nextInterval * 24 * 60 * 60 * 1000),
      intervalDay: nextInterval,
      ease: Math.max(1.3, baseEase - 0.05),
      streak: 0,
      reviews,
    }
  }

  const nextInterval = current.intervalDay <= 0
    ? 1
    : Math.max(1, Math.round(current.intervalDay * baseEase))
  return {
    ...current,
    dueAt: new Date(now.getTime() + nextInterval * 24 * 60 * 60 * 1000),
    intervalDay: nextInterval,
    ease: Math.min(2.8, baseEase + 0.05),
    streak: current.streak + 1,
    reviews,
  }
}
```

### 3.2 设备识别模块

本系统不使用传统用户登录，而是通过设备 ID 识别用户。

```
┌─────────────────────────────────────────────────────────────┐
│                     设备识别流程                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  首次请求                                                    │
│     │                                                       │
│     ├─── 检查 localStorage 是否有设备ID                     │
│     │         │                                             │
│     │         ├─── 无 ──→ 生成新设备ID                      │
│     │         │         dev_${timestamp}_${random}          │
│     │         │              │                              │
│     │         │              └──→ 存入 localStorage         │
│     │         │                                              │
│     │         └─── 有 ──→ 使用已有设备ID                     │
│     │                                                       │
│     └──→ 发送请求时携带 x-device-id 请求头                   │
│                                                             │
│  后端处理                                                    │
│     │                                                       │
│     ├─── 检查 x-device-id 请求头                            │
│     │         │                                             │
│     │         ├─── 无 ──→ 生成新设备ID，响应头返回           │
│     │         │                                             │
│     │         └─── 有 ──→ 使用请求头中的设备ID               │
│     │                                                       │
│     └──→ 根据 deviceId 查询/创建用户学习状态                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 虚拟列表模块

为解决大量单词渲染性能问题，使用虚拟列表技术。

```

│                     虚拟列表原理                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │

│  ┌─────────────────────┐                                    │
│  │  Item 3  (渲染)     │                                    │
│  │  Item 4  (渲染)     │                                    │
│  │  Item 5  (渲染)     │                                    │

│  └─────────────────────┘                                    │
│                                                             │

│  ┌─────────────────────┐                                    │

│  │  Item 3  (渲染)     │ ← 可视区域                         │

│                                                             │

│  ├─── 传统渲染：10000 个 DOM 节点                           │

│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 6. 性能优化

### 6.1 前端优化

| 优化项 | 方案 | 效果 |
|--------|------|------|
